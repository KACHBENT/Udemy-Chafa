¿Qué es el modelo, vista y controlador (MVC)?

Es un patrón arquitectónico de software que divide una aplicación en tres componentes interconectados: el modelo, la vista y el controlador

Este patrón tiene como objetivo principal la separación de responsabilidades, facilitando el desarrollo, mantenimiento y escalabilidad de las aplicaciones.

> [!IMPORTANT]
> Es esencial que el avance del proyecto cuente con conocimientos y habilidades necesarias, así como con  las certificaciones, para poder programar siguiendo buenas prácticas y crear un código que sea organizado y que se pueda ampliar.
